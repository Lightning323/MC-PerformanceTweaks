## Optimizations
- Skips collision checks on the client's side because these checks are already handled by the server, reducing the strain on your client when there are lots of entities present.
- Can reduce server side lag by reducing number of block updates and power calculations made to Redstone.
- Replace Minecraft's expensive and wasteful recursive method calling with a much faster check by a single powered rail. Credit to FX_PR0CESS: (Curseforge), (Github)
